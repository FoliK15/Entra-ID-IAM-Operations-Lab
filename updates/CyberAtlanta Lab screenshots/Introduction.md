# CYBER AD LAB (Cyber Atlanta) — IAM Training Environment

> **Update Track:** Hands-on Active Directory & IAM workflow simulation  
> **Goal:** Build practical IAM operations experience through realistic tickets, access requests, governance tasks, and security monitoring.

## Overview
This section documents my progress in the **Cyber Atlanta IAM Training Lab**, a simulated Active Directory/IAM environment designed to replicate common identity operations and administrative workflows. The artifacts below highlight completed lab paths, operational tasks performed (tickets, access requests, offboarding), and supporting audit activity.

> **Note:** This is a simulated training environment (not a production tenant). Screenshots and evidence are included to demonstrate workflow familiarity and repeatable IAM operations.

---

## What This Update Demonstrates
- **Identity lifecycle operations** (onboarding/offboarding, password resets, unlocks)
- **Group & access management workflows** (security groups, membership changes, access requests)
- **Security & governance fundamentals** (password policies, audit log review, privileged groups)
- **Operational ticket handling** (service desk workflow / request completion)
- **Monitoring & alert response** (reviewing and resolving security alerts)
- **Architecture & automation exposure** (OU structure concepts + automation job configuration)

## Key Takeaways
- IAM work is built around **repeatable workflows**: request → verification → change → documentation → audit evidence.
- Good IAM execution requires strong habits in **validation** (audit activity), **least privilege** (privileged groups), and **process discipline** (ticket-based work).
- These lab paths strengthen operational readiness for entry IAM roles such as **IAM Analyst**, **Access Management Analyst**, or **Identity Support**.
